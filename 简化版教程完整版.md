你有一个静态住宅ip，如何让电脑里访问特定域名时走这个ip呢？下面主包给大家带来 clash配置链式代理访问chatgpt 相关域名的教程
0️⃣以clash verge为例，p2展示了在“订阅-你的订阅-右键”弹出的选项框，后续的选择都是在这里选择

![IMG_20260317_232459](D:\下载/IMG_20260317_232459.png)

1️⃣选择“编辑节点-右上角高级”，在弹出的代码栏里，如下替换为你的信息。

```yaml
prepend:
  - name: 节点名1 #随便取名字，但是要和下一步编辑规则里的名字对应
    type: socks5 
    server: 你的静态住宅IP 
    port: 你的端口 
    username: 你的用户名 
    password: 你的密码 
    dialer-proxy: 'xxxxx' #这里填你当前订阅里真实存在的代理组名字，必须一字不差；如果组名里有空格、emoji，建议加单引号

append: []

delete: []
```

2️⃣选择“编辑规则-右上角高级”，在弹出的代码栏里，如下替换为你的信息，这里的匹配规则是openai相关域名的，如果你是其他的域名，可以让ai帮你修改。

```yaml
prepend:
  - DOMAIN-KEYWORD,openai,节点名1
  - DOMAIN-SUFFIX,chatgpt.com,节点名1
  - DOMAIN-SUFFIX,oaistatic.com,节点名1
  - DOMAIN-SUFFIX,oaiusercontent.com,节点名1
  - DOMAIN-SUFFIX,featuregates.org,节点名1
  - DOMAIN-SUFFIX,statsig.com,节点名1 #（包含这条）以上是域名匹配规则，凡是匹配到的都走“节点名1”这个节点，这里你可以让ai帮忙生成匹配规则，你只需要告诉它你节点名字，需要和上一步编辑节点里的名字对应
  - DOMAIN,api.ipify.org,节点名1 #这个网站是测试ip的，方便我们测试是否成功。

append: []

delete: []
```

✈️配完后的检查顺序

1. 第一步中 dialer-proxy中你选择的代理组中，选择一个稳定的海外节点。
2. 确认规则里的目标节点名与上面添加的节点名完全一致（例如都叫节点名1 ）
3. 打开  `https://api.ipify.org`，看看是否返回你的静态住宅ip，是的话就成功了。

✅原理很简单，第一步我们设置了一个节点，这个节点的是一个链式节点，经过它的流量会先走dialer-proxy里设置的海外节点，然后再跳到你的静态住宅ip。第二步我们添加了匹配规则，凡是匹配到我们设置的规则的域名时，都走我们第一步的链式代理，这样你访问的网站，例如我这里的openai，看到的就是静态住宅的ip，而不是梯子的ip。
